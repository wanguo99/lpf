### 10.2 单元测试

#### 10.2.1 测试范围

| 层级 | 测试模块 | 用例数 | 覆盖功能 |
|------|---------|-------|---------|
| OSAL | 版本/原子/字符串/时间/内存/日志/文件/互斥锁/信号量/条件变量 | 50+ | 系统抽象接口 |
| HAL  | CAN/串口/I2C/SPI | 72 | 硬件驱动接口 |
| PCL  | 配置查询/平台管理 | 5+ | 配置管理 |
| PDL  | Satellite/BMC/MCU/看门狗 | 15+ | 外设服务 |

#### 10.2.2 测试特点

- ✅ OSAL/PCL层：无硬件依赖，可在任何平台运行
- ⚠️  HAL层：依赖虚拟硬件（vcan0、/dev/ttyUSB0）或实际硬件
- ⚠️  PDL层：部分测试依赖外部设备（BMC服务器、MCU）

#### 10.2.3 测试框架

```c
/* 统一测试框架 */
TEST_CASE(test_osal_mutex_create)
{
    osal_id_t mutex_id;
    int32_t ret = OSAL_MutexCreate(&mutex_id, "test", 0);
    TEST_ASSERT_EQUAL(OS_SUCCESS, ret);
    OSAL_MutexDelete(mutex_id);
}

TEST_MODULE_BEGIN(test_osal_mutex, "互斥锁测试")
    TEST_CASE_REGISTER(test_osal_mutex_create, "互斥锁创建")
    TEST_CASE_REGISTER(test_osal_mutex_lock, "互斥锁加锁")
TEST_MODULE_END()
```

**运行方式**：

```bash
# 运行所有测试
./build/bin/ems-test -a

# 运行指定层级
./build/bin/osal-test -a    # OSAL层
./build/bin/hal-test -a     # HAL层

# 交互式菜单
./build/bin/ems-test -i
```

### 10.3 系统测试

#### 10.3.1 测试场景

| 测试场景 | 验证点 | 性能目标 |
|---------|-------|---------|
| 遥控命令流程 | 卫星平台→Telecommand→ACL→PDL→响应 | <2ms |
| 缓存型遥测流程 | 后台采集→共享内存→快速响应 | <200μs |
| 实时型遥测流程 | 实时查询→超时降级→响应 | <1.2ms |
| 遥控后遥测流程 | 缓存失效→STALE标记 | 数据新鲜度保证 |
| 故障恢复流程 | 进程崩溃→检测→重启→恢复 | 5秒内恢复 |
| BMC双通道切换 | 主通道故障→自动切换备用通道 | 服务不中断 |

#### 10.3.2 实时性能验证

| 测试项 | 目标 | 验证方法 |
|-------|------|---------|
| 遥控命令延迟 | <2ms | 高精度时间戳测量（1000次采样） |
| 缓存型遥测延迟 | <200μs | 共享内存读取延迟测量 |
| 实时型遥测延迟 | <1.2ms | PDL查询+降级延迟测量 |
| CPU占用 | <60% | 正常负载下CPU使用率监控 |
| 内存占用 | <50MB | 进程内存使用监控 |

#### 10.3.3 故障恢复验证

| 故障类型 | 注入方式 | 预期恢复行为 |
|---------|---------|-------------|
| 进程崩溃 | kill -9 进程 | Supervisor检测并重启（5秒内） |
| 硬件故障 | 断开CAN/网络 | 降级运行，切换备用通道 |
| 共享内存损坏 | 随机翻转位（SEU） | CRC校验失败，重新初始化 |
| 内存耗尽 | 持续分配内存 | 内存监控告警，拒绝新分配 |

### 10.4 压力测试

#### 10.4.1 测试配置

| 测试类型 | 配置 | 验证点 |
|---------|------|-------|
| 长时间稳定性 | 7×24小时运行，遥控10次/秒 | 无崩溃、无内存泄漏、无资源耗尽 |
| 高负载并发 | 遥控100次/秒，持续10分钟 | 响应延迟<2ms，无丢包 |
| 突发负载 | 1秒内1000次遥控 | 队列不溢出，降级处理 |
| 资源泄漏检测 | 1小时持续运行 | 内存/FD/线程无泄漏 |
| 故障注入 | 每小时1次随机故障 | 自动恢复，服务连续性 |

#### 10.4.2 监控指标

资源监控（每分钟采样）：
- CPU使用率
- 内存占用（堆/栈）
- 文件描述符数量
- 线程数量
- 队列深度

性能监控：
- 响应延迟（Min/Avg/P50/P95/P99/Max）
- 吞吐量（命令/秒）
- 错误率（超时/失败）

### 10.6 测试环境

#### 10.6.1 开发环境

- **平台**：Ubuntu 22.04 LTS (x86_64)
- **虚拟硬件**：vcan0（虚拟CAN）、socat（虚拟串口）
- **测试工具**：GDB、Valgrind、strace、perf

```bash
# 设置虚拟硬件
sudo modprobe vcan
sudo ip link add dev vcan0 type vcan
sudo ip link set vcan0 up
```

#### 10.6.2 目标硬件

- **SoC**：TI AM6254 (ARM Cortex-A53, 单核 @ 1.4GHz)
- **内存**：512MB DDR4
- **接口**：CAN×2、Ethernet×2、UART×4

```bash
# 交叉编译并部署
./build.sh -a arm64
scp -r build/arm64/bin/* root@192.168.1.100:/opt/ems/bin/
ssh root@192.168.1.100 "cd /opt/ems/bin && ./ems-test -a"
```
