## 🔴 Critical 级别问题（必须立即修复）

### C-1: HAL层直接使用系统调用，绕过OSAL

**问题描述**：
HAL层所有Linux实现都直接调用系统调用（open, close, read, write, ioctl等），完全绕过OSAL抽象层。

**位置**：
- `core/hal/src/linux/hal_i2c_linux.c`
- `core/hal/src/linux/hal_serial_linux.c`
- `core/hal/src/linux/hal_can_linux.c`
- `core/hal/src/linux/hal_spi_linux.c`
- `core/hal/src/linux/hal_gpio_linux.c`

**违反原则**：原则2（可移植性）

**影响**：
- 严重违反可移植性原则
- 无法移植到RTOS平台
- 代码与Linux强耦合
- 统计：80+ 处直接系统调用

**修复建议**：
1. 在OSAL中添加文件和设备操作接口：
```c
// core/osal/include/io/osal_file.h
int32_t OSAL_FileOpen(const char *path, int32_t flags, osal_fd_t *fd);
int32_t OSAL_FileClose(osal_fd_t fd);
int32_t OSAL_FileRead(osal_fd_t fd, void *buf, size_t count, ssize_t *nread);
int32_t OSAL_FileWrite(osal_fd_t fd, const void *buf, size_t count, ssize_t *nwritten);
int32_t OSAL_FileIoctl(osal_fd_t fd, uint32_t request, void *arg);
```

2. 修改HAL层使用OSAL接口：
```c
// 当前（违反原则2）
int fd = open(config->device, O_RDWR);

// 修复后
osal_fd_t fd;
OSAL_FileOpen(config->device, OSAL_O_RDWR, &fd);
```

**工作量**：10-15天
- OSAL接口设计和实现：5天
- HAL层重构：5-7天
- 测试验证：3天

**优先级**：P0（最高优先级）

**前置依赖**：无

**验收标准**：
- HAL层代码中无任何直接系统调用
- 所有系统调用通过OSAL封装
- 通过静态分析工具验证（grep "open\|close\|read\|write\|ioctl" core/hal/）

### C-2: PCL依赖PDL形成循环依赖

**问题描述**：
配置层（PCL）依赖驱动层（PDL）的头文件，违反分层原则，形成循环依赖。

**位置**：
- `core/pcl/CMakeLists.txt:4`
- `core/pcl/include/pcl_mcu.h`

**违反原则**：原则1（模块解耦）

**影响**：
- 破坏模块独立性
- 增加编译依赖
- 无法单独编译PCL
- 违反Clean Architecture原则

**修复建议**：
反转依赖方向，让PDL依赖PCL：

```cmake
# 当前（错误）
# core/pcl/CMakeLists.txt
target_link_libraries(pcl PUBLIC pdl)

# 修复后
# core/pdl/CMakeLists.txt
target_link_libraries(pdl PUBLIC pcl)
```

重新设计PCL配置结构，不依赖PDL类型：
```c
// pcl_mcu.h（不依赖PDL）
typedef struct {
    const char *name;
    const char *device_path;
    uint32_t    baudrate;
    // ... 其他配置字段
} pcl_mcu_config_t;
```

**工作量**：5-7天
- 重新设计PCL配置结构：2天
- 更新PDL层引用PCL：3天
- 迁移产品层配置：2天

**优先级**：P0

**前置依赖**：无

**验收标准**：
- PCL不依赖PDL
- PDL依赖PCL
- 依赖关系图验证无循环

**违反原则**：原则1（模块解耦）

```

│   └── tm_cache.h
├── src/
│   └── tm_cache.c
├── CMakeLists.txt
```

```c
// tm_cache.h
int32_t TM_Cache_Init(size_t max_entries);

int32_t TM_Cache_Put(uint32_t tm_id, const void *data, size_t len);
```

**工作量**：3-5天

**优先级**：P0

### C-6: PRL CRC验证动态内存分配

**问题描述**：
每次验证CRC都分配内存，性能差（10x慢）且可能失败。

**位置**：
- `core/prl/src/prl_common.c:131`

**违反原则**：原则3（可维护性）

**影响**：
- 性能瓶颈（比分段计算慢10倍）
- 内存分配失败风险
- 不适合实时系统

**修复建议**：
使用分段计算CRC，避免拷贝和动态分配：

```c
// 当前（动态分配）
uint8_t *temp = OSAL_Malloc(total_len);
OSAL_Memcpy(temp, packet, total_len);
((prl_header_t *)temp)->crc16 = 0;
uint16_t crc = prl_calc_crc16(temp, total_len);
OSAL_Free(temp);

// 修复后（分段计算）
uint16_t crc = prl_calc_crc16(packet, offsetof(prl_header_t, crc16));
uint16_t zero = 0;
crc = prl_calc_crc16_update(crc, (uint8_t*)&zero, sizeof(zero));
crc = prl_calc_crc16_update(crc, 
    packet + offsetof(prl_header_t, crc16) + sizeof(uint16_t),
    total_len - offsetof(prl_header_t, crc16) - sizeof(uint16_t));
```

**工作量**：2-3天
- 实现分段CRC计算：1天
- 重构验证函数：1天
- 性能测试和验证：1天

**优先级**：P0

**前置依赖**：无

**验收标准**：
- 无动态内存分配
- 性能提升10倍
- 功能测试通过

```c
union {

thread_union.posix_thread = pt;
*thread = thread_union.osal_thread;

// 修复后
typedef pthread_t osal_thread_t;  // 直接使用pthread_t
```

**工作量**：1天

**优先级**：P0

**前置依赖**：无

**验收标准**：

### C-9: ACL全局状态无线程保护

**问题描述**：
全局变量`g_acl_table`无互斥锁保护，多线程不安全。

**位置**：
- `core/acl/src/acl_api.c:10`

**违反原则**：原则3（可维护性）

**影响**：
- 数据竞争
- 崩溃风险
- 多线程环境不可用

**修复建议**：
```c
static const acl_config_table_t *g_acl_table = NULL;
static osal_rwlock_t *g_acl_rwlock = NULL;

int32_t ACL_Init(void) {
    return OSAL_RWLockCreate(&g_acl_rwlock);
}

int32_t ACL_RegisterTable(const acl_config_table_t *table) {
    OSAL_RWLockWriteLock(g_acl_rwlock);
    g_acl_table = table;
    OSAL_RWLockWriteUnlock(g_acl_rwlock);
    return OSAL_SUCCESS;
}

const acl_tc_config_t* ACL_GetTcConfig(uint32_t function_id) {
    OSAL_RWLockReadLock(g_acl_rwlock);
    const acl_tc_config_t *config = NULL;
    if (g_acl_table && function_id < g_acl_table->tc_count) {
        config = &g_acl_table->tc_table[function_id];
    }
    OSAL_RWLockReadUnlock(g_acl_rwlock);
    return config;
}
```

**工作量**：2-3天

**优先级**：P0

**前置依赖**：无

**验收标准**：
- 所有全局状态访问有锁保护
- 线程安全测试通过
