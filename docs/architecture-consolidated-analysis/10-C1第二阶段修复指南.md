## 修改清单

### 1. 静态互斥锁处理

**问题（第35行）：**
```c
static pthread_mutex_t gpio_isr_mutex = PTHREAD_MUTEX_INITIALIZER;
```

**解决方案：**
```c
static osal_mutex_t *gpio_isr_mutex = NULL;

// 添加模块初始化函数
static int32_t gpio_module_init(void) {
    if (gpio_isr_mutex == NULL) {
        if (OSAL_SUCCESS != OSAL_MutexCreate(&gpio_isr_mutex)) {
            LOG_ERROR("HAL_GPIO", "Failed to create ISR mutex");
            return OSAL_ERR_NO_MEMORY;
        }
    }
    return OSAL_SUCCESS;
}

// 添加模块清理函数
static void gpio_module_cleanup(void) {
    if (gpio_isr_mutex != NULL) {
        OSAL_MutexDelete(gpio_isr_mutex);
        gpio_isr_mutex = NULL;
    }
}
```

**调用点修改（10处）：**
- 第230行：`pthread_mutex_lock(&gpio_isr_mutex)` → `OSAL_MutexLock(gpio_isr_mutex)`
- 第233行：`pthread_mutex_unlock(&gpio_isr_mutex)` → `OSAL_MutexUnlock(gpio_isr_mutex)`
- 第246行、249行、262行、265行、278行、281行、294行、297行：类似修改

**初始化调用：**
在第一次使用GPIO ISR功能时调用`gpio_module_init()`

### 3. 线程操作修改

#### 3.1 线程创建（第380行）

**原代码：**

## 注意事项

### 1. 静态初始化问题

**问题**：pthread_mutex可以静态初始化，OSAL_Mutex不行

**解决**：
- 使用模块初始化函数
- 在第一次使用前调用
- 使用`__attribute__((constructor))`确保早期初始化（可选）

### 2. 线程取消

**问题**：pthread_cancel在OSAL中未实现

**解决**：
- 使用标志位（running）实现优雅退出
- 不要强制取消线程
- 在线程函数中定期检查running标志

### 3. 错误处理

**确保**：
- 检查所有OSAL函数的返回值
- 在错误路径中正确清理资源
- 保持原有的错误处理逻辑

### 4. 线程安全

**保持**：
- 不要改变加锁的范围
- 不要改变加锁的顺序
- 保持原有的线程安全逻辑
