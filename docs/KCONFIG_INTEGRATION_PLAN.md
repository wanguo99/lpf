# Kconfig + Kbuild Integration Plan for EMS Project

## Context

The user wants to integrate a Kconfig + Kbuild configuration system **directly into the EMS project**. The goal is to provide a Linux kernel-style configuration interface (`make menuconfig`, `make defconfig`, etc.) that generates `.config` and `include/generated/autoconf.h` for use in C source code.

**Current State:**
- EMS uses pure CMake build system with hardcoded configuration headers
- No existing Kconfig infrastructure
- Three-layer configuration: PCL (peripheral config), ACL (application config), and hardcoded headers
- Multi-architecture support (x86_64, ARM32, ARM64, RISC-V)
- Build options scattered across CMakeLists.txt files (BUILD_TESTING, BUILD_PMC_*, etc.)

**Why Integrate Kconfig:**
- Provide user-friendly configuration interface (menuconfig)
- Single source of truth for build-time configuration
- Industry-standard approach (Linux kernel, Buildroot, U-Boot)
- Better dependency management between features
- Replace hardcoded config headers (uart_config.h, can_config.h) with generated ones
- Easier to maintain than scattered CMake options

**Approach:**
We'll integrate Kconfig **directly into EMS** while keeping CMake as the primary build system. The workflow will be:
1. User runs `make menuconfig` to configure features
2. Kconfig generates `.config` and `include/generated/autoconf.h`
3. CMake reads the generated configuration and builds accordingly
4. This creates a hybrid Kconfig + CMake system (similar to U-Boot)

### 2.2 Top-Level Makefile (Kconfig Wrapper)

**File:** `/home/wanguo/EMS/Makefile`

```makefile
# EMS Kconfig Wrapper Makefile
# This Makefile handles Kconfig operations only.
# Actual building is done by CMake (via build.sh or direct cmake commands).

# Kconfig configuration
KCONFIG_CONFIG := .config
export KCONFIG_CONFIG

CONF := scripts/kconfig/conf
MCONF := scripts/kconfig/mconf

# Directories
INCLUDE_DIR := include
CONFIG_DIR := $(INCLUDE_DIR)/config
GENERATED_DIR := $(INCLUDE_DIR)/generated

# ============================================================================
# Kconfig Targets
# ============================================================================

.PHONY: menuconfig config defconfig savedefconfig oldconfig silentoldconfig syncconfig

menuconfig: $(MCONF)
	@mkdir -p $(CONFIG_DIR) $(GENERATED_DIR)
	@echo "Starting menuconfig..."
	@$< Kconfig
	@$(MAKE) syncconfig

config: $(CONF)
	@mkdir -p $(CONFIG_DIR) $(GENERATED_DIR)
	@$< --oldaskconfig Kconfig
	@$(MAKE) syncconfig

defconfig: $(CONF)
	@mkdir -p $(CONFIG_DIR) $(GENERATED_DIR)
	@echo "Loading default configuration..."
	@$< --defconfig=defconfig Kconfig
	@$(MAKE) syncconfig

%_defconfig: configs/%_defconfig $(CONF)
	@mkdir -p $(CONFIG_DIR) $(GENERATED_DIR)
	@echo "Loading configuration: $*"
	@$(CONF) --defconfig=$< Kconfig
	@$(MAKE) syncconfig

savedefconfig: $(CONF)
	@echo "Saving minimal configuration to defconfig..."
	@$< --savedefconfig=defconfig Kconfig
	@echo "Configuration saved."

oldconfig: $(CONF)
	@mkdir -p $(CONFIG_DIR) $(GENERATED_DIR)
	@$< --oldconfig Kconfig
	@$(MAKE) syncconfig

silentoldconfig: syncconfig

syncconfig: $(CONF)
	@mkdir -p $(CONFIG_DIR) $(GENERATED_DIR)
	@$< --syncconfig Kconfig
	@echo "Configuration synchronized."

# ============================================================================
# Build Kconfig Tools
# ============================================================================

$(CONF) $(MCONF):
	@echo "Building kconfig tools..."
	@$(MAKE) -C scripts/kconfig $(notdir $@)

# ============================================================================
# CMake Integration Targets
# ============================================================================

.PHONY: cmake-configure cmake-build build rebuild

cmake-configure: syncconfig
	@echo "Configuring CMake with Kconfig settings..."
	@if [ ! -f .config ]; then \
		echo "Error: No .config found. Run 'make menuconfig' or 'make defconfig' first."; \
		exit 1; \
	fi
	@./build.sh

cmake-build: cmake-configure
	@echo "Building with CMake..."
	@cmake --build build -j$$(nproc)

build: cmake-build

rebuild: distclean defconfig build

# ============================================================================
# Clean Targets
# ============================================================================

.PHONY: clean-config distclean

clean-config:
	@echo "Cleaning Kconfig artifacts..."
	@rm -f .config .config.old
	@rm -rf $(CONFIG_DIR) $(GENERATED_DIR)

distclean: clean-config
	@echo "Cleaning kconfig tools..."
	@$(MAKE) -C scripts/kconfig clean

# ============================================================================
# Help
# ============================================================================

.PHONY: help

help:
	@echo "EMS Kconfig Configuration System"
	@echo ""
	@echo "Configuration targets:"
	@echo "  menuconfig       - Interactive configuration (ncurses)"
	@echo "  defconfig        - Load default configuration"
	@echo "  <name>_defconfig - Load preset from configs/<name>_defconfig"
	@echo "  savedefconfig    - Save minimal config to defconfig"
	@echo "  oldconfig        - Update config with new options"
	@echo "  syncconfig       - Synchronize and generate headers"
	@echo ""
	@echo "Build targets (Kconfig + CMake):"
	@echo "  build            - Configure and build with CMake"
	@echo "  cmake-configure  - Run CMake configuration"
	@echo "  cmake-build      - Build with CMake"
	@echo "  rebuild          - Clean config, load defaults, and build"
	@echo ""
	@echo "Clean targets:"
	@echo "  clean-config     - Remove .config and generated headers"
	@echo "  distclean        - Remove config and kconfig tools"
	@echo ""
	@echo "Typical workflow:"
	@echo "  1. make menuconfig      # Configure features"
	@echo "  2. make build           # Build with CMake"
	@echo "  or"
	@echo "  1. make defconfig       # Use defaults"
	@echo "  2. ./build.sh           # Build directly"
	@echo ""
	@echo "Available presets:"
	@ls -1 configs/*_defconfig 2>/dev/null | sed 's|configs/||' | sed 's|^|  |' || echo "  (none)"
```

```makefile
# Standalone Kconfig tools Makefile

# Host compiler
HOSTCC := gcc
HOSTCXX := g++

# ncurses flags
NCURSES_CFLAGS := $(shell pkg-config --cflags ncurses 2>/dev/null || echo)
NCURSES_LIBS := $(shell pkg-config --libs ncurses 2>/dev/null || echo -lncurses)

# Targets
hostprogs := conf mconf nconf

# conf (command-line tool)
conf-objs := conf.o zconf.tab.o

# mconf (menuconfig - ncurses)
mconf-objs := mconf.o zconf.tab.o lxdialog/checklist.o lxdialog/inputbox.o \
              lxdialog/menubox.o lxdialog/textbox.o lxdialog/util.o lxdialog/yesno.o

# nconf (newer ncurses interface)
nconf-objs := nconf.o zconf.tab.o nconf.gui.o

# Build rules
all: conf mconf

conf: $(conf-objs)
	$(HOSTCC) -o $@ $^

mconf: $(mconf-objs)
	$(HOSTCC) -o $@ $^ $(NCURSES_LIBS)

nconf: $(nconf-objs)
	$(HOSTCC) -o $@ $^ $(NCURSES_LIBS) -lmenu -lpanel

# Object compilation
%.o: %.c
	$(HOSTCC) $(NCURSES_CFLAGS) -c -o $@ $<

lxdialog/%.o: lxdialog/%.c
	$(HOSTCC) $(NCURSES_CFLAGS) -c -o $@ $<

# Parser generation (if flex/bison available)
zconf.tab.c: zconf.y
	bison -o $@ -t -l $<

zconf.lex.c: zconf.l
	flex -o $@ -L $<

# Clean
clean:
	rm -f conf mconf nconf *.o lxdialog/*.o zconf.tab.c zconf.lex.c

.PHONY: all clean
```

```cmake
# Include generated configuration
```

### 2.5 Configuration Generation Logic

Kconfig automatically generates configuration files when you run `make syncconfig` (or `silentoldconfig` in older versions).

**Generated Files:**

1. **`.config`** - Human-readable configuration (key=value format)
2. **`include/config/auto.conf`** - Makefile-compatible variables
3. **`include/generated/autoconf.h`** - C preprocessor defines

**Generation Process:**

```bash
# User configures via menuconfig
make menuconfig

# Kconfig tools generate files
scripts/kconfig/conf --syncconfig Kconfig
```

## 3. Recommended Standalone Kconfig Base

**Primary Recommendation:** Extract from Linux kernel (most authentic, production-ready)

**Repository:** https://cdn.kernel.org/pub/linux/kernel/v6.x/linux-6.1.92.tar.xz

**Why Linux kernel kconfig:**
- ✅ Industry standard (used by Linux, U-Boot, Buildroot, Zephyr)
- ✅ Battle-tested and stable
- ✅ Full feature set (dependencies, choices, ranges, etc.)
- ✅ Native C implementation (no Python dependency)
- ✅ ncurses-based menuconfig (familiar interface)

**Alternative:** Kconfiglib (Python-based)
- Repository: https://github.com/ulfalizer/Kconfiglib
- Pros: Easier to install (pip install), pure Python, good documentation
- Cons: Requires Python runtime, slightly different behavior from kernel kconfig

**Integration Steps (Linux kernel kconfig):**

```bash
cd /home/wanguo/EMS

# 1. Download Linux kernel source
wget https://cdn.kernel.org/pub/linux/kernel/v6.x/linux-6.1.92.tar.xz
tar -xf linux-6.1.92.tar.xz

# 2. Copy kconfig infrastructure
mkdir -p scripts/kconfig
cp -r linux-6.1.92/scripts/kconfig/* scripts/kconfig/

# 3. Copy required headers (if needed)
mkdir -p scripts/include
cp -r linux-6.1.92/scripts/include/* scripts/include/ 2>/dev/null || true

# 4. Clean up
rm -rf linux-6.1.92 linux-6.1.92.tar.xz

# 5. Create simplified Makefile for kconfig tools
cat > scripts/kconfig/Makefile << 'EOF'
# Standalone Kconfig Tools Makefile

HOSTCC := gcc
HOSTCXX := g++

# ncurses configuration
NCURSES_CFLAGS := $(shell pkg-config --cflags ncurses 2>/dev/null)
NCURSES_LIBS := $(shell pkg-config --libs ncurses 2>/dev/null || echo -lncurses)

# Targets
.PHONY: all conf mconf nconf clean

all: conf mconf

# conf - command-line configuration tool
conf-objs := conf.o zconf.tab.o
conf: $(conf-objs)
	$(HOSTCC) -o $@ $^

# mconf - menuconfig (ncurses interface)
mconf-objs := mconf.o zconf.tab.o lxdialog/checklist.o lxdialog/inputbox.o \
              lxdialog/menubox.o lxdialog/textbox.o lxdialog/util.o lxdialog/yesno.o
mconf: $(mconf-objs)
	$(HOSTCC) -o $@ $^ $(NCURSES_LIBS)

# nconf - newer ncurses interface (optional)
nconf-objs := nconf.o zconf.tab.o nconf.gui.o
nconf: $(nconf-objs)
	$(HOSTCC) -o $@ $^ $(NCURSES_LIBS) -lmenu -lpanel

# Compilation rules
%.o: %.c
	$(HOSTCC) $(NCURSES_CFLAGS) -I. -c -o $@ $<

lxdialog/%.o: lxdialog/%.c
	$(HOSTCC) $(NCURSES_CFLAGS) -I. -c -o $@ $<

# Parser generation (requires flex/bison)
zconf.tab.c: zconf.y
	bison -o $@ -t -l $<

zconf.lex.c: zconf.l
	flex -o $@ -L $<

# Clean
clean:
	rm -f conf mconf nconf *.o lxdialog/*.o
	rm -f zconf.tab.c zconf.lex.c zconf.hash.c
EOF

# 6. Install build dependencies
sudo apt-get update
sudo apt-get install -y build-essential libncurses-dev flex bison pkg-config

# 7. Build kconfig tools
cd scripts/kconfig
make conf mconf
cd ../..

# 8. Verify installation
./scripts/kconfig/conf --help
./scripts/kconfig/mconf --help

echo "Kconfig tools installed successfully!"
```

```cmake
    message(WARNING "Using default configuration...")

# Parse Kconfig settings (helper function)
function(kconfig_is_set option result_var)
    if(EXISTS ${KCONFIG_HEADER})
        file(STRINGS ${KCONFIG_HEADER} CONFIG_LINES)
        if("${CONFIG_LINES}" MATCHES "#define ${option}")
            set(${result_var} TRUE PARENT_SCOPE)
        else()
            set(${result_var} FALSE PARENT_SCOPE)
        endif()
    else()
        set(${result_var} FALSE PARENT_SCOPE)
    endif()
endfunction()

# Read build type from Kconfig
kconfig_is_set(CONFIG_BUILD_TYPE_DEBUG IS_DEBUG)
if(IS_DEBUG)
    set(CMAKE_BUILD_TYPE Debug)
    message(STATUS "Build type: Debug (from Kconfig)")
```

**In C Files:**

**In CMakeLists.txt:**

```cmake
# Read Kconfig configuration
set(KCONFIG_HEADER "${CMAKE_SOURCE_DIR}/include/generated/autoconf.h")

# Helper function to check if option is set
function(kconfig_is_set option result_var)
    if(EXISTS ${KCONFIG_HEADER})
        file(STRINGS ${KCONFIG_HEADER} CONFIG_LINES)
        if("${CONFIG_LINES}" MATCHES "#define ${option}")
            set(${result_var} TRUE PARENT_SCOPE)
        else()
            set(${result_var} FALSE PARENT_SCOPE)
        endif()
    else()
        set(${result_var} FALSE PARENT_SCOPE)
    endif()
endfunction()

# Helper function to get option value
function(kconfig_get_value option result_var)
    if(EXISTS ${KCONFIG_HEADER})
        file(STRINGS ${KCONFIG_HEADER} CONFIG_LINES)
        foreach(line ${CONFIG_LINES})
            if("${line}" MATCHES "#define ${option} (.+)")
                set(${result_var} ${CMAKE_MATCH_1} PARENT_SCOPE)
                return()
            endif()
        endforeach()
    endif()
    set(${result_var} "" PARENT_SCOPE)
endfunction()

# Use in CMake
kconfig_is_set(CONFIG_BUILD_TESTING BUILD_TESTS)
if(BUILD_TESTS)
    enable_testing()
    add_subdirectory(tests)
endif()
```

### 6.3 Lightweight Design

**Minimal Footprint:**

```
Component                Size        Required
-------------------------------------------------
conf (CLI tool)          ~50 KB      Yes
mconf (menuconfig)       ~150 KB     Yes
nconf (optional)         ~200 KB     No
.config                  ~5-10 KB    Yes (generated)
autoconf.h               ~10-20 KB   Yes (generated)
auto.conf                ~5-10 KB    Yes (generated)
Kconfig files            ~20-50 KB   Yes (source)
-------------------------------------------------
Total (minimal)          ~240 KB     
Total (with sources)     ~300 KB
```

**Optimization Tips:**

1. **Use only conf and mconf** - Skip nconf, gconf, qconf
2. **Minimize Kconfig files** - Keep configuration schema concise
3. **Use choice instead of multiple bools** - Reduces generated code
4. **Group related options** - Better organization, smaller files
5. **Use depends on** - Prevents invalid configurations

**Comparison with Alternatives:**

```
Configuration System     Footprint   Complexity   Features
---------------------------------------------------------
Kconfig + mconf          ~300 KB     Medium       Full
Kconfiglib (Python)      ~500 KB     Low          Full
CMake options only       0 KB        Low          Basic
Custom config parser     ~50 KB      High         Limited
```

## 12. Success Criteria

The Kconfig integration is considered successful when:

1. ✅ **Functional Requirements:**
   - `make menuconfig` launches interactive configuration
   - `make defconfig` loads default configuration
   - `make savedefconfig` saves minimal configuration
   - Configuration generates `.config` and `autoconf.h`
   - CMake reads and uses generated configuration
   - Conditional compilation works based on CONFIG_* options

2. ✅ **Quality Requirements:**
   - All existing functionality preserved
   - Build time not significantly increased
   - Configuration is intuitive and well-documented
   - Dependencies between options work correctly
   - Cross-compilation still works

3. ✅ **Documentation Requirements:**
   - Quick start guide available
   - All configuration options documented
   - Migration guide for existing code
   - Troubleshooting section complete

4. ✅ **Testing Requirements:**
   - All presets load correctly
   - CMake integration tested
   - Cross-compilation tested
   - Automated test suite passes
