## Architecture Overview

### Design Goals

- **Single Source of Truth**: Kconfig `.config` file drives both CMake and C compilation
- **Automatic Synchronization**: Configuration changes automatically propagate to build system
- **Native CMake Integration**: No runtime dependency on Python for configuration loading
- **Standard Tooling**: Compatible with standard Kconfig tools (menuconfig, defconfig)

### System Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                     Configuration Layer                          │
│                                                                   │
│  Kconfig Files          defconfig Files        .config           │
│  (menu structure)       (presets)              (active config)   │
└─────────────────────────────────────────────────────────────────┘
                                  │
                                  ├── make ──┐_defconfig
                                  │                              │
                                  ▼                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                     Generation Layer                             │
│                                                                   │
│  tools/kconfig/genconfig.py                                      │
│  (parses .config, generates output files)                        │
└─────────────────────────────────────────────────────────────────┘
                                  │
                    ┌─────────────┴──────────────┐
                    ▼                            ▼
        ┌────────────────────┐      ┌────────────────────┐
        │  kconfig.cmake     │      │  autoconf.h        │
        │  (CMake variables) │      │  (C macros)        │
        └────────────────────┘      └────────────────────┘
                    │                            │
                    ▼                            ▼
        ┌────────────────────┐      ┌────────────────────┐
        │  CMakeLists.txt    │      │  C Source Files    │
        │  (conditional      │      │  (conditional      │
        │   compilation)     │      │   compilation)     │
        └────────────────────┘      └────────────────────┘
                    │                            │
                    └─────────────┬──────────────┘
                                  ▼
                        ┌──────────────────┐
                        │  Build Artifacts │
                        │  (_build/)       │
                        └──────────────────┘
```

### Key Components

| Component | Purpose | Generated By | Used By |
|-----------|---------|--------------|---------|
| `.config` | Active configuration | `python3 build.py config` | `genconfig.py` |
| `kconfig.cmake` | CMake variables | `genconfig.py` | CMake |
| `autoconf.h` | C macro definitions | `genconfig.py` | C compiler |
| `cmake/Kconfig.cmake` | Integration module | Manual | Top-level CMakeLists.txt |

## Configuration Flow

### Step 1: Load Configuration

```bash
# Load predefined configuration
make ccm_h200_100p_am625_debug_defconfig_defconfig

# Or use menuconfig for interactive configuration
make menuconfig
```

**What happens:**
1. `build.py config` copies `configs/<defconfig>` to `.config`
2. Kconfig resolves dependencies and applies defaults
3. `.config` contains the final resolved configuration

### Step 2: Generate Build Files

When you run CMake (via `make` or manually):

```cmake
# In CMakeLists.txt
include(cmake/Kconfig.cmake)
kconfig_load()
```

**What happens:**
1. CMake executes `tools/kconfig/genconfig.py`
2. Script parses `.config` and generates:
   - `_build/kconfig.cmake` - CMake variables
   - `_build/autoconf.h` - C macro definitions
3. CMake includes `kconfig.cmake` making variables available
4. CMake configures `-include autoconf.h` for all compilations

### Step 3: Conditional Compilation

**In CMakeLists.txt:**
```cmake
if(CONFIG_OSAL)
    add_subdirectory(core/osal)
endif()
```

**In C code:**
```c
#ifdef CONFIG_OSAL_DEBUG
    printf("Debug enabled\n");
#endif
```

## CMake Integration

### The `kconfig_load()` Function

Located in `cmake/Kconfig.cmake`, this function is the core of the integration.

#### Usage

```cmake
cmake_minimum_required(VERSION 3.16)
project(ES-Middleware C)

# Load Kconfig integration module
include(cmake/Kconfig.cmake)

# Load configuration and generate files
kconfig_load()

# Now CONFIG_* variables are available
if(CONFIG_BUILD_TESTING)
    enable_testing()
endif()
```

#### Implementation Details

```cmake
function(kconfig_load)
    # 1. Check for .config file
    if(NOT EXISTS "${CMAKE_SOURCE_DIR}/.config")
        message(FATAL_ERROR "No .config found. Run: make <defconfig>")_defconfig
    endif()

    # 2. Generate output files
    execute_process(
        COMMAND ${Python3_EXECUTABLE}
                ${CMAKE_SOURCE_DIR}/tools/kconfig/genconfig.py
                ${CMAKE_SOURCE_DIR}/.config
                ${CMAKE_BINARY_DIR}/kconfig.cmake
                ${CMAKE_BINARY_DIR}/autoconf.h
        RESULT_VARIABLE result
    )
    
    if(NOT result EQUAL 0)
        message(FATAL_ERROR "Failed to generate kconfig.cmake and autoconf.h")
    endif()

    # 3. Load CMake variables
    include(${CMAKE_BINARY_DIR}/kconfig.cmake)

    # 4. Configure autoconf.h auto-include
    add_compile_options(-include ${CMAKE_BINARY_DIR}/autoconf.h)
endfunction()
```

### Generated File Formats

#### kconfig.cmake Format

```cmake
# Generated from .config - DO NOT EDIT

# Boolean options
set(CONFIG_OSAL ON)
set(CONFIG_HAL ON)
set(CONFIG_BUILD_TESTING OFF)

# Integer options
set(CONFIG_MAX_THREADS 16)

# String options
set(CONFIG_PLATFORM_NAME "H200-100P-AM625")

# Hex options
set(CONFIG_BASE_ADDRESS 0x80000000)
```

#### autoconf.h Format

```c
/* Generated from .config - DO NOT EDIT */
#ifndef AUTOCONF_H
#define AUTOCONF_H

/* Boolean options - defined as 1 or not defined */
#define CONFIG_OSAL 1
#define CONFIG_HAL 1
/* CONFIG_BUILD_TESTING is not set */

/* Integer options */
#define CONFIG_MAX_THREADS 16

/* String options */
#define CONFIG_PLATFORM_NAME "H200-100P-AM625"

/* Hex options */
#define CONFIG_BASE_ADDRESS 0x80000000

#endif /* AUTOCONF_H */
```

### Using Configuration Variables in CMake

#### Conditional Compilation

```cmake
# Add subdirectory if module enabled
if(CONFIG_OSAL)
    add_subdirectory(core/osal)
endif()

# Conditional source files
if(CONFIG_HAL_CAN)
    target_sources(hal PRIVATE src/hal_can.c)
endif()

# Conditional dependencies
if(CONFIG_USE_EXTERNAL_LIB)
    find_package(ExternalLib REQUIRED)
    target_link_libraries(myapp PRIVATE ExternalLib::ExternalLib)
endif()
```

#### Configuration-Based Settings

```cmake
# Use string configuration
set(TARGET_PLATFORM ${CONFIG_PLATFORM_NAME})
message(STATUS "Building for platform: ${TARGET_PLATFORM}")

# Use integer configuration
if(CONFIG_MAX_THREADS GREATER 32)
    message(WARNING "Thread pool size may be excessive: ${CONFIG_MAX_THREADS}")
endif()

# Use hex configuration
target_compile_definitions(myapp PRIVATE
    BASE_ADDR=${CONFIG_BASE_ADDRESS}
)
```

Utility function for debugging configuration issues.


## C Code Integration

### Automatic Header Inclusion

The `autoconf.h` header is automatically included in **all** C source files via the `-include` compiler flag. You don't need to manually include it.

```c
/* No need to #include "autoconf.h" - it's automatic */

void my_function(void)
{
    #ifdef CONFIG_OSAL_DEBUG
        printf("Debug mode\n");
    #endif
}
```

### Conditional Compilation Patterns

#### Feature Gates

```c
#ifdef CONFIG_FEATURE_X
    /* Feature X implementation */
    void feature_x_init(void) {
        // ...
    }
#else
    /* Stub implementation */
    void feature_x_init(void) {
        /* Feature X disabled */
    }
#endif
```

#### Debug Logging

```c
void critical_function(void)
{
    #ifdef CONFIG_DEBUG_VERBOSE
        printf("[DEBUG] Entering critical_function\n");
    #endif
    
    /* Function implementation */
    
    #ifdef CONFIG_DEBUG_VERBOSE
        printf("[DEBUG] Exiting critical_function\n");
    #endif
}
```

#### Platform-Specific Code

```c
#if defined(CONFIG_ARCH_ARM64)
    #include <arm_neon.h>
    void optimized_operation(void) {
        /* ARM64 NEON implementation */
    }
#elif defined(CONFIG_ARCH_X86_64)
    #include <immintrin.h>
    void optimized_operation(void) {
        /* x86_64 SSE/AVX implementation */
    }
#else
    void optimized_operation(void) {
        /* Generic implementation */
    }
#endif
```

#### Configuration Value Usage

```c
/* Integer configuration */
#define THREAD_POOL_SIZE CONFIG_MAX_THREADS
static pthread_t threads[THREAD_POOL_SIZE];

/* String configuration */
const char *platform_name = CONFIG_PLATFORM_NAME;

/* Compile-time checks */
#if CONFIG_MAX_THREADS > 64
    #error "Thread pool size exceeds maximum (64)"
#endif
```

### Best Practices for C Code

1. **Use `#ifdef` for boolean options:**
   ```c
   #ifdef CONFIG_FEATURE_X
   ```
   Not `#if CONFIG_FEATURE_X` (works but less clear)

2. **Check configuration value ranges:**
   ```c
   #if CONFIG_BUFFER_SIZE < 256
       #error "Buffer size too small"
   #endif
   ```

3. **Provide fallback implementations:**
   ```c
   #ifdef CONFIG_HARDWARE_ACCELERATION
       void process(void) { /* HW accelerated */ }
   #else
       void process(void) { /* Software fallback */ }
   #endif
   ```

4. **Document configuration dependencies:**
   ```c
   /* This function requires CONFIG_CAN_BUS=y */
   #ifdef CONFIG_CAN_BUS
   int can_send_message(uint32_t id, const uint8_t *data, size_t len);
   #endif
   ```

## Developer Workflow

### Scenario 1: Adding a New Module

**Goal:** Add a new `core/newmodule` library with Kconfig integration.

#### Step 1: Create Kconfig File

`core/newmodule/Kconfig`:
```kconfig
config NEWMODULE
    bool "Enable New Module"
    default y
    help
      This is a new module that does X, Y, Z.

if NEWMODULE

config NEWMODULE_DEBUG
    bool "Enable debug logging"
    default n

config NEWMODULE_BUFFER_SIZE
    int "Buffer size"
    default 1024
    range 256 4096

endif # NEWMODULE
```

#### Step 2: Update Parent Kconfig

`core/Kconfig`:
```kconfig
source "core/osal/Kconfig"
source "core/hal/Kconfig"
source "core/newmodule/Kconfig"  # Add this line
```

#### Step 3: Create CMakeLists.txt

`core/newmodule/CMakeLists.txt`:
```cmake
# Only build if enabled in Kconfig
if(NOT CONFIG_NEWMODULE)
    return()
endif()

add_library(newmodule
    src/newmodule.c
    src/newmodule_utils.c
)

target_include_directories(newmodule
    PUBLIC ${CMAKE_CURRENT_SOURCE_DIR}/include
)

# Configuration-dependent sources
if(CONFIG_NEWMODULE_DEBUG)
    target_sources(newmodule PRIVATE src/debug.c)
endif()
```

#### Step 4: Update Parent CMakeLists.txt

`core/CMakeLists.txt`:
```cmake
add_subdirectory(osal)
add_subdirectory(hal)
add_subdirectory(newmodule)  # Add this line
```

#### Step 5: Use in C Code

`core/newmodule/src/newmodule.c`:
```c
#include "newmodule.h"

void newmodule_init(void)
{
    #ifdef CONFIG_NEWMODULE_DEBUG
        printf("Initializing newmodule with buffer size: %d\n",
               CONFIG_NEWMODULE_BUFFER_SIZE);
    #endif
    
    /* Initialization code */
}
```

#### Step 6: Update Configuration and Build

```bash
# Update configuration to enable new module
make menuconfig
# Navigate to "Core Modules" -> "Enable New Module" -> Y

# Or create a new defconfig
make savedefconfig my_config_defconfig

# Build
make
```

### Scenario 2: Creating a Custom Configuration

**Goal:** Create a minimal configuration for a specific use case.

```bash
# 1. Start with closest existing config
make tests_x86_minimal_defconfig_defconfig

# 2. Customize via menuconfig
make menuconfig
# Disable unwanted modules
# Adjust buffer sizes, thread counts, etc.

# 3. Save as new defconfig
make savedefconfig my_minimal_defconfig
# This creates configs/my_minimal_defconfig

# 4. Test the configuration
make distclean
make my_minimal_defconfig_defconfig
make

# 5. Commit to repository
git add configs/my_minimal_defconfig
git commit -m "feat: add minimal configuration for use case X"
```

### Scenario 3: Debugging Configuration Issues

**Problem:** Module not building even though CONFIG option is enabled.

```bash
# 1. Verify .config has the option
grep CONFIG_MY_MODULE .config
# Should show: CONFIG_MY_MODULE=y

# 2. Check generated CMake variables
cat _build/kconfig.cmake | grep CONFIG_MY_MODULE
# Should show: set(CONFIG_MY_MODULE ON)

# 3. Check generated C macros
cat _build/autoconf.h | grep CONFIG_MY_MODULE
# Should show: #define CONFIG_MY_MODULE 1

# 4. Verify CMakeLists.txt logic
# Check if the if(CONFIG_MY_MODULE) block is correct

# 5. Force CMake reconfiguration
make distclean
make <defconfig>_defconfig
make --verbose
```

## Advanced Topics

### Configuration Dependencies

#### Automatic Dependency Resolution

Kconfig automatically handles dependencies:

```kconfig
# In core/pdl/Kconfig
config PDL
    bool "Peripheral Driver Layer"
    depends on HAL  # PDL requires HAL
    default y

# In core/aconfig/Kconfig
config ACONFIG
    bool "Application Configuration Layer"
    select PCONFIG  # ACONFIG automatically enables PCONFIG
    default y
```

**Behavior:**
- `depends on`: Option only selectable if dependency is met
- `select`: Automatically enables dependency when option is selected

#### Handling Symbol Dependencies in CMake

```cmake
# PDL depends on HAL - CMake should enforce this
if(CONFIG_PDL AND NOT CONFIG_HAL)
    message(FATAL_ERROR "PDL requires HAL. Enable CONFIG_HAL or disable CONFIG_PDL.")
endif()

# Or let Kconfig handle it (preferred)
if(CONFIG_PDL)
    # Kconfig already ensures HAL is enabled
    add_subdirectory(core/pdl)
endif()
```

### Cross-Platform Configuration

#### Platform-Specific Options

```kconfig
choice
    prompt "Target Architecture"
    default ARCH_X86_64

config ARCH_X86_64
    bool "x86_64"

config ARCH_ARM64
    bool "ARM64 (aarch64)"

endchoice

# Platform-specific features
if ARCH_ARM64

config ARM_NEON
    bool "Enable ARM NEON optimizations"
    default y

endif # ARCH_ARM64
```

#### Platform-Specific Compilation

```cmake
# In CMakeLists.txt
if(CONFIG_ARCH_ARM64)
    set(CMAKE_C_COMPILER aarch64-linux-gnu-gcc)
    set(CMAKE_CXX_COMPILER aarch64-linux-gnu-g++)
    
    if(CONFIG_ARM_NEON)
        add_compile_options(-mfpu=neon)
    endif()
elseif(CONFIG_ARCH_X86_64)
    # Native compilation
endif()
```

### Conditional Testing

```cmake
if(CONFIG_BUILD_TESTING)
    enable_testing()
    
    # Add tests based on enabled modules
    if(CONFIG_OSAL)
        add_subdirectory(tests/core/osal)
    endif()
    
    if(CONFIG_PDL)
        add_subdirectory(tests/core/pdl)
    endif()
endif()
```

### Optimizing Build Time

#### Parallel Configuration Generation

Currently, `genconfig.py` is fast (<1 second). For very large projects, consider:

```cmake
# Cache genconfig.py output
set(KCONFIG_TIMESTAMP ${CMAKE_BINARY_DIR}/kconfig.timestamp)

if(${CMAKE_SOURCE_DIR}/.config IS_NEWER_THAN ${KCONFIG_TIMESTAMP})
    # Regenerate only when .config changes
    execute_process(COMMAND python3 tools/kconfig/genconfig.py ...)
    file(TOUCH ${KCONFIG_TIMESTAMP})
else()
    message(STATUS "Kconfig outputs up-to-date")
endif()
```

#### Minimal Reconfiguration

```bash
# Only reconfigure CMake, don't rebuild everything
cmake _build/

# Or use --fresh flag (CMake 3.24+)
cmake --fresh -B _build/
```

## Troubleshooting

### Common Issues and Solutions

#### Issue 1: "No .config found"

**Error:**
```
CMake Error: No .config found. Run: make <defconfig>_defconfig
```

**Solution:**
```bash
make ccm_h200_100p_am625_debug_defconfig_defconfig
```

#### Issue 2: CONFIG_* Variable Not Defined in CMake

**Symptoms:**
- `if(CONFIG_XXX)` always evaluates to false
- Module not building even though enabled in .config

**Diagnosis:**
```bash
# Check .config
grep CONFIG_XXX .config

# Check generated kconfig.cmake
cat _build/kconfig.cmake | grep CONFIG_XXX

# Check CMake variables
cmake -B _build/ -LA | grep CONFIG_XXX
```

**Solution:**
1. Ensure `kconfig_load()` is called in top-level CMakeLists.txt
2. Clear CMake cache: `rm -rf _build/CMakeCache.txt _build/CMakeFiles`
3. Reconfigure: `make`

#### Issue 3: CONFIG_* Macro Not Defined in C Code

**Symptoms:**
- Compiler warnings: `CONFIG_XXX` is not defined
- Conditional compilation not working

**Diagnosis:**
```bash
# Check autoconf.h
cat _build/autoconf.h | grep CONFIG_XXX

# Check compile commands
make -- VERBOSE=1 | grep "include.*autoconf.h"
```

**Solution:**
1. Verify autoconf.h exists and contains the macro
2. Ensure `kconfig_load()` is called (it sets up `-include`)
3. Check if CMakeLists.txt overrides compile options

#### Issue 4: Configuration Changes Not Applied

**Symptoms:**
- Modified .config but build output unchanged
- defconfig loaded but symbols missing

**Diagnosis:**
```bash
# Check if CMake cache is stale
ls -l _build/CMakeCache.txt .config

# Compare timestamps
stat _build/kconfig.cmake .config
```

**Solution:**
```bash
# Force full reconfiguration
make distclean
make <defconfig>_defconfig
make
```

#### Issue 5: genconfig.py Fails

**Error:**
```
CMake Error: genconfig.py exited with code 1
```

**Diagnosis:**
```bash
# Run manually to see error
python3 tools/kconfig/genconfig.py .config _build/kconfig.cmake _build/autoconf.h

# Check Python version
python3 --version
```

**Solution:**
1. Ensure Python 3.6+ is installed
2. Check .config format (may be corrupted)
3. Restore from defconfig: `make <defconfig>`_defconfig

#### Issue 6: Symbol Dependencies Not Satisfied

**Symptoms:**
- Enabled CONFIG_XXX but it disappears after loading defconfig
- menuconfig shows option grayed out

**Diagnosis:**
```bash
# Use menuconfig to inspect dependencies
make menuconfig
# Navigate to symbol, press '?'
```

**Solution:**
1. Check Kconfig `depends on` clauses
2. Enable required parent options
3. Update defconfig to include dependencies

### Debug Checklist

When configuration isn't working:

- [ ] `.config` exists and contains expected symbols
- [ ] `_build/kconfig.cmake` exists and was generated recently
- [ ] `_build/autoconf.h` exists and contains expected macros
- [ ] Top-level CMakeLists.txt calls `kconfig_load()`
- [ ] CMake cache is not stale (delete `_build/CMakeCache.txt` if unsure)
- [ ] Symbol dependencies are satisfied in Kconfig
- [ ] No CMakeLists.txt syntax errors preventing configuration

### Getting Help

If issues persist:

1. Run with verbose output:
   ```bash
   make --verbose 2>&1 | tee build.log
   ```

2. Check generated files:
   ```bash
   ls -la .config _build/kconfig.cmake _build/autoconf.h
   ```

3. Compare with working configuration:
   ```bash
   diff .config configs/tests_x86_full_defconfig
   ```

4. Review CMake configuration output for errors

## Summary

The Kconfig + CMake integration provides:

✅ **Single source of truth** - `.config` drives everything  
✅ **Automatic synchronization** - Changes propagate to build system  
✅ **Native CMake integration** - No Python dependency at build time  
✅ **Standard tooling** - Compatible with menuconfig and defconfig  
✅ **Type-safe variables** - CMake and C see consistent configuration  

**Key Takeaways:**

1. Always load configuration before building: `make <defconfig>`_defconfig
2. Use `kconfig_load()` in top-level CMakeLists.txt
3. Configuration changes require CMake reconfiguration
4. `autoconf.h` is automatically included in all C files
5. When in doubt, `distclean` and reconfigure
