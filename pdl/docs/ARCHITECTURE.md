# PDL 架构设计

## 设计理念

PDL (Peripheral Driver Layer) 采用**外设为中心**的设计，将卫星平台、BMC载荷、MCU等设备统一抽象为"外设"，提供统一的服务接口。

## 核心概念

### 外设抽象

在EMS中，管理板是核心，其他设备都是外设：
- **卫星平台** → 外设（通过CAN/1553B通信）
- **BMC载荷** → 外设（通过以太网/串口通信）
- **MCU** → 外设（通过CAN/UART/I2C/SPI通信）

## 三大外设服务

### 1. 卫星平台服务

**架构**：
```
卫星平台 <--CAN--> PDL Satellite <--> 应用层
```

**关键特性**：
- **心跳机制**：5秒周期心跳检测
- **命令处理**：接收卫星平台命令，回调通知应用
- **遥测上报**：定期上报遥测数据
- **统计信息**：收发计数、错误计数

**数据结构**：
```c
typedef struct {
    const char *can_device;
    uint32_t can_bitrate;
    uint32_t heartbeat_interval_ms;
    satellite_cmd_callback_t cmd_callback;
} satellite_pdl_config_t;
```

### 2. BMC载荷服务

**架构**：
```
BMC载荷 <--以太网(主)--> PDL BMC <--> 应用层
         <--串口(备)-->
```

**关键特性**：
- **双通道冗余**：以太网（主）+ 串口（备）
- **自动故障切换**：5次连续失败触发切换
- **协议支持**：IPMI over LAN、Redfish
- **电源管理**：开机、关机、重启、状态查询

**故障切换逻辑**：
```
主通道正常 → 使用主通道
主通道失败 → 失败计数++
失败计数 >= 5 → 切换到备通道
备通道成功 → 重置失败计数
备通道失败 → 失败计数++
失败计数 >= 5 → 切换回主通道
```

### 3. MCU外设服务

**架构**：
```
MCU <--CAN/UART/I2C/SPI--> PDL MCU <--> 应用层
```

**关键特性**：
- **多接口支持**：CAN、UART、I2C、SPI
- **版本查询**：读取MCU固件版本
- **固件升级**：支持固件在线升级
- **GPIO控制**：复位、中断等GPIO操作

## 平台无关性

PDL层必须保持完全平台无关：
- ✅ 使用HAL接口访问硬件（`HAL_CAN_Send()`, `HAL_Serial_Send()`）
- ✅ 使用OSAL接口进行系统调用（`OSAL_TaskCreate()`, `OSAL_Malloc()`）
- ❌ 禁止包含任何系统头文件（`<unistd.h>`, `<sys/socket.h>`等）
- ❌ 禁止直接调用系统API

## 任务模型

每个外设服务通常包含以下任务：

### 卫星平台服务任务
```
1. 接收任务：监听CAN总线，接收命令
2. 心跳任务：定期发送心跳帧
3. 遥测任务：定期上报遥测数据
```

### BMC载荷服务任务
```
1. 监控任务：监控主通道健康状态
2. 故障切换任务：检测故障并切换通道
```
